# Implement (구현)

설계 문서에 **정확히** 따라 코드를 구현하세요.

## 핵심 원칙

### 설계 준수
- 설계 문서에 명시된 것만 구현
- 설계에 없는 "개선"이나 "추가 기능" 금지
- 설계와 다르게 해야 할 경우 → design 단계로 돌아가기

### 최소 변경
- 요구사항 해결에 필요한 최소한의 코드만 작성
- 관련 없는 리팩토링 금지
- "ついでに" 정리하기 금지

### 품질 유지
- 기존 코드 스타일 준수
- 새 경고/에러 도입 금지
- 기존 테스트 깨지면 안됨

## 필수 수행 사항

### 1. 구현 전 확인
- [ ] 설계 문서의 파일 변경 계획 확인
- [ ] 각 파일의 변경 규모 확인
- [ ] 의존성 순서 파악

### 2. 구현 순서
1. 의존성이 없는 것부터 시작
2. 한 번에 하나의 파일/기능 집중
3. 작은 단위로 동작 확인

### 3. 테스트
- 설계의 테스트 전략에 따라 테스트 작성
- 기존 테스트 실행하여 회귀 확인
- 수용 기준 충족 확인

### 4. 검증
- [ ] 모든 수용 기준 충족
- [ ] 기존 테스트 통과
- [ ] 새 테스트 통과
- [ ] 빌드 성공
- [ ] 린트 에러 없음

## 금지 사항

- 설계에 없는 파일 수정
- 설계에 없는 기능 추가
- "더 나은" 방법으로 임의 변경
- 관련 없는 코드 정리/리팩토링
- 주석/문서 임의 추가
- 설계 변경 없이 다른 접근법 사용

## 예외 상황

설계대로 구현이 불가능한 경우:
1. 구현 중단
2. 문제점 문서화
3. design 단계로 돌아가서 설계 수정

## 완료 조건

- [ ] 설계 문서의 모든 파일 변경 완료
- [ ] 모든 수용 기준 충족
- [ ] 모든 테스트 통과
- [ ] 빌드 성공
- [ ] 설계에 없는 변경 없음
