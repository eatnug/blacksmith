# Constitution

프로젝트 전역에서 적용되는 원칙입니다. 모든 스텝에서 이 문서를 참조합니다.

---

## 불명확 처리 원칙

정보가 부족하거나 여러 해석이 가능한 경우:

1. **추측하지 않는다**
2. **`[NEEDS CLARIFICATION: 구체적 질문]` 마커를 붙인다**
3. **마커가 있는 상태로 다음 단계 진행 불가**

### 예시

```markdown
## 요구사항
- 사용자 인증 기능 구현
- 소셜 로그인 지원 [NEEDS CLARIFICATION: Google만 지원? Apple도 필요?]
- 세션 유지 [NEEDS CLARIFICATION: 유지 시간은 얼마로?]
```

### 마커 해결

사용자가 답변하면 마커를 제거하고 내용을 반영합니다:

```markdown
## 요구사항
- 사용자 인증 기능 구현
- 소셜 로그인 지원 (Google, Apple)
- 세션 유지 (7일)
```

---

## 프로젝트 원칙

> 아래는 예시입니다. 프로젝트에 맞게 수정하세요.

### 예시 원칙

```markdown
## 프로젝트 원칙

- 테스트 없이 구현 없다
- 기존 인터페이스 breaking change 금지
- 외부 의존성 추가 시 사유 문서화 필수
```

---

## 사용법

이 파일을 프로젝트에 맞게 수정하세요:

1. "불명확 처리 원칙"은 유지 (권장)
2. "프로젝트 원칙" 섹션에 팀 규칙 추가
3. 필요 없는 예시는 삭제
