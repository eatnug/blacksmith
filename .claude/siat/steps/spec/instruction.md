# Spec (요구사항 명세)

사용자의 요청을 분석하여 **구체적이고 측정 가능한 요구사항**으로 변환하세요.

## 필수 수행 사항

### 1. 요청 분석
- 사용자가 **무엇을** 원하는지 파악
- **왜** 필요한지 (목적, 문제점) 이해
- **누가** 사용하는지 (사용자/시스템) 확인

### 2. 불명확한 부분 식별
- 모호하거나 해석이 여러 가지인 부분을 **반드시** 질문
- 가정하지 말고 물어보세요
- 예: "삭제 시 확인 다이얼로그가 필요한가요?" "에러 발생 시 어떻게 처리할까요?"

### 3. 수용 기준 작성
- 각 요구사항에 대해 "완료"의 정의를 명확히
- Given-When-Then 또는 체크리스트 형태
- 테스트 가능해야 함

### 4. 범위 정의
- 이번에 하는 것 (In Scope)
- 이번에 하지 않는 것 (Out of Scope)
- 경계가 명확해야 함

## 금지 사항

- 기술적 구현 방법 논의 (design 단계에서)
- 코드 작성 (implement 단계에서)
- 불명확한 채로 넘어가기

## 완료 조건

- [ ] 모든 요구사항이 구체적이고 측정 가능함
- [ ] 수용 기준이 각 요구사항에 정의됨
- [ ] 범위가 명확히 정의됨
- [ ] 열린 질문이 모두 해결됨 (또는 문서화됨)
