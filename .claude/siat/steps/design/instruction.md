# Design (기술 설계)

요구사항을 **구현 가능한 기술 설계**로 변환하세요.

## 필수 수행 사항

### 1. 코드베이스 분석
- 기존 아키텍처와 패턴 파악
- 관련 코드 위치 확인
- 기존 컨벤션 이해

### 2. 설계 결정
- 각 요구사항을 어떻게 구현할지 결정
- 여러 접근법이 있다면 비교하고 선택 근거 문서화
- 트레이드오프 명시

### 3. 인터페이스 정의
- 함수/메서드 시그니처
- API 엔드포인트 (해당시)
- 컴포넌트 props/인터페이스

### 4. 파일 구조 설계
- 수정할 파일 목록
- 새로 생성할 파일 목록
- 삭제할 파일 목록 (해당시)

### 5. 의존성 분석
- 영향받는 다른 모듈/컴포넌트
- 필요한 외부 라이브러리 (해당시)
- Breaking changes 여부

## 금지 사항

- 요구사항 변경 (spec으로 돌아가세요)
- 실제 코드 작성 (implement 단계에서)
- 설계 없이 바로 구현하기

## 설계 체크리스트

- [ ] 모든 요구사항(FR)에 대한 구현 방법이 정의됨
- [ ] 기존 코드 패턴과 일관성 유지
- [ ] 인터페이스가 명확히 정의됨
- [ ] 변경 파일 목록이 완성됨
- [ ] 영향도 분석 완료
- [ ] 기술적 결정에 근거가 있음

## 완료 조건

- [ ] implement 단계에서 추가 결정 없이 바로 코딩 가능
- [ ] 다른 개발자가 읽고 구현할 수 있는 수준
